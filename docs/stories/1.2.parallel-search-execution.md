## Story
**As a** research pipeline,
**I want** to execute all Tavily searches concurrently with rate limiting,
**so that** the 55–60 API calls complete in minimal time without hitting API limits.

## Acceptance Criteria
1. LangGraph node `execute_searches(state: ResearchState) -> ResearchState` in `app/graph/nodes.py`.
2. Uses `AsyncTavilyClient` for async search execution.
3. `asyncio.Semaphore(5)` limits concurrent searches to maximum 5 at any moment.
4. Each search call uses: `max_results=5`, `search_depth="advanced"`, `include_raw_content=False`.
5. Results stored in `state.raw_results` as `dict[tuple[str, str, int], list[dict]]` — key = (vendor, req_id, query_index).
6. Individual search failures are caught with `try/except`, logged, and stored as empty list for that pair (do not abort pipeline).
7. Progress events emitted every 5 searches completed (update state for SSE).
8. Total wall clock time for search phase logged.

## Dev Notes
**Depends on**: Story 1.1 (query generation, state.queries), Story 2.1 (ResearchState).

**AsyncTavilyClient pattern** (from planning docs):
```python
from tavily import AsyncTavilyClient
import asyncio

async def execute_searches(state: ResearchState) -> ResearchState:
    client = AsyncTavilyClient(api_key=settings.TAVILY_API_KEY)
    semaphore = asyncio.Semaphore(5)

    async def search_with_limit(vendor: str, req_id: str, query_idx: int, query: str):
        async with semaphore:
            try:
                results = await client.search(
                    query=query,
                    max_results=5,
                    search_depth="advanced",
                    include_raw_content=False
                )
                return (vendor, req_id, query_idx), results.get("results", [])
            except Exception as e:
                logging.error(f"Search failed for {vendor}×{req_id}: {e}")
                return (vendor, req_id, query_idx), []

    tasks = []
    for (vendor, req_id), (q1, q2) in state["queries"].items():
        tasks.append(search_with_limit(vendor, req_id, 0, q1))
        tasks.append(search_with_limit(vendor, req_id, 1, q2))

    results = await asyncio.gather(*tasks)
    state["raw_results"] = {k: v for k, v in results}
    return state
```

**Tavily result structure**: each result dict has `url`, `content`, `title`, `score` (relevance from Tavily), `published_date`.

**Total searches for iteration 1**: 4 vendors × 6 requirements × 2 queries = 48 searches. Iteration 2 (gap filling): 5–10 additional. Total: ~55–60.

**Time filter**: use `days=365` parameter in Tavily search to prefer recent content.
