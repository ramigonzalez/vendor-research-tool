## Story
**As a** research pipeline,
**I want** to extract structured `Evidence` objects from raw Tavily search results using an LLM,
**so that** each piece of evidence has traceable metadata that feeds the scoring engine.

## Acceptance Criteria
1. LangGraph node `extract_evidence(state: ResearchState) -> ResearchState` in `app/graph/nodes.py`.
2. For each vendorĂ—requirement pair, combines search results from both queries and sends to LLM.
3. LLM extracts `Evidence` objects with all required fields: `claim`, `source_url`, `source_name`, `source_type`, `content_date`, `relevance` (0â€“1), `supports_requirement` (bool).
4. `source_type` classified as: `official_docs`, `github`, `comparison`, `blog`, `community` based on URL and content.
5. LLM instructed conservatively: only extract explicitly stated claims, not inferences.
6. Multiple `Evidence` objects may be extracted per search result if it contains multiple claims.
7. Evidence stored in `state.evidence` as `dict[tuple[str, str], list[Evidence]]` keyed by (vendor, req_id).
8. Evidence also persisted to SQLite via repository (bulk INSERT per pair).
9. Empty evidence list acceptable â€” gap analysis handles sparse results in Story 1.4.
10. LLM calls for all 24 pairs run concurrently.

## Dev Notes
**Depends on**: Story 1.2 (raw_results), Story 2.1 (Evidence model), Story 3.2 (repository).

**Evidence extraction prompt** (from planning docs):
```
You are extracting evidence about a vendor's capabilities from search results.

Vendor: {vendor}
Requirement: {requirement} (Priority: {priority})

Search Results:
{formatted_results}

Extract specific, concrete evidence claims. For each claim:
- Write the exact claim as stated in the source (no paraphrasing)
- Classify source_type: official_docs, github, comparison, blog, community
- Rate relevance 0.0â€“1.0 (1.0 = directly addresses requirement)
- Set supports_requirement: true if vendor clearly has this capability

BE CONSERVATIVE. If ambiguous, set supports_requirement=false and relevance < 0.5.
Return JSON: {"evidence": [{"claim": "...", "source_url": "...", "source_name": "...", "source_type": "...", "content_date": "...", "relevance": 0.8, "supports_requirement": true}]}
```

**Content truncation**: cap each Tavily result `content` field at 2000 characters before sending to LLM to avoid context overflow.

**Source type classification heuristics** (provide to LLM in prompt):
- Contains `.github.com` or `github.com/` â†’ `github`
- Contains `docs.`, `/docs/`, or official product domain â†’ `official_docs`
- Contains "vs", "comparison", "alternative" â†’ `comparison`
- Medium.com, dev.to, personal blogs â†’ `blog`
- Reddit, HN, Discord, Slack communities â†’ `community`

**Concurrent extraction**: `asyncio.gather` for all 24 pairs. Each pair may have 2â€“10 search results.

**Repository call**: after extraction, call `await repo.save_evidence(job_id, vendor, req_id, evidence_list)`.
