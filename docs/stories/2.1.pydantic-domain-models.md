## Story
**As a** development team,
**I want** all domain objects defined as typed Pydantic v2 models in `app/models.py`,
**so that** every data boundary in the system is type-safe, validated, and self-documenting.

## Acceptance Criteria
1. `SourceType` enum defined: `official_docs`, `github`, `comparison`, `blog`, `community`.
2. `CapabilityLevel` enum defined: `full`, `partial`, `minimal`, `none`, `unknown`.
3. `MaturityLevel` enum defined: `ga`, `beta`, `experimental`, `planned`, `unknown`.
4. `JobStatus` enum defined: `pending`, `running`, `completed`, `failed`.
5. `Evidence` Pydantic model: `claim: str`, `source_url: str`, `source_name: str`, `source_type: SourceType`, `content_date: str | None = None`, `relevance: float` (0–1, validated), `supports_requirement: bool`.
6. `LLMAssessment` Pydantic model: `capability_level: CapabilityLevel`, `capability_details: str`, `maturity: MaturityLevel`, `limitations: list[str]`, `supports_requirement: bool`.
7. `ScoreResult` Pydantic model: `score: float` (0–10), `confidence: float` (0–1), `capability_level: CapabilityLevel`, `maturity: MaturityLevel`, `justification: str`, `limitations: list[str]`, `evidence: list[Evidence]`.
8. `ResearchJob` Pydantic model: `id: str`, `status: JobStatus`, `created_at: datetime`, `completed_at: datetime | None = None`, `progress_pct: int = 0`, `progress_message: str = ""`.
9. `Requirement` Pydantic model: `id: str`, `description: str`, `priority: Priority` (enum: `high`, `medium`, `low`). No per-requirement `weight` field — weights are derived from priority category via `config.PRIORITY_WEIGHTS` (High=3.0, Medium=2.0, Low=1.0).
10. `VendorRanking` Pydantic model: `vendor: str`, `overall_score: float`, `rank: int`.
11. `ResearchResults` Pydantic model: `vendors: list[str]`, `requirements: list[Requirement]`, `matrix: dict[str, dict[str, ScoreResult]]`, `rankings: list[VendorRanking]`, `summary: str`.
12. `ResearchState` TypedDict (LangGraph): `job_id`, `vendors`, `requirements`, `queries`, `raw_results`, `evidence`, `assessments`, `scores`, `rankings`, `summary`, `iteration`, `gaps`, `progress_queue`.
13. All models in `app/models.py`. All pass `mypy --strict` or `pyright` type checking.

## Dev Notes
**Critical path**: This story must be completed FIRST before any other story. All other stories import from `app/models.py`.

**Tech Stack**:
- Python 3.11+
- Pydantic v2 (`from pydantic import BaseModel, field_validator`)
- LangGraph `TypedDict` from `typing_extensions`

**Key Pydantic v2 patterns**:
```python
from pydantic import BaseModel, field_validator
from enum import Enum
from datetime import datetime

class SourceType(str, Enum):
    official_docs = "official_docs"
    github = "github"
    # ...

class Evidence(BaseModel):
    relevance: float

    @field_validator('relevance')
    @classmethod
    def validate_relevance(cls, v: float) -> float:
        return max(0.0, min(1.0, v))
```

**ResearchState** lives in `app/graph/state.py` as a `TypedDict` (not Pydantic) — LangGraph requires TypedDict for state.

**`matrix` field type**: `dict[str, dict[str, ScoreResult]]` — outer key = vendor, inner key = requirement_id.
