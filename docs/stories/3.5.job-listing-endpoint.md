## Story
**As a** frontend,
**I want** to list previous research jobs,
**so that** users can navigate to past research results.

## Acceptance Criteria
1. `GET /api/jobs` returns `list[ResearchJob]` as JSON array.
2. Jobs sorted by `created_at` descending (most recent first).
3. Maximum 50 jobs returned.
4. Empty list `[]` returned when no jobs exist.
5. Each job includes: `id`, `status`, `created_at`, `completed_at`, `progress_pct`.

## Tasks / Subtasks
- [ ] Implement `GET /api/jobs` route in router
  - [ ] Call `repo.list_jobs(limit=50)`
  - [ ] Return list serialized as JSON
- [ ] Write unit test

## Dev Notes
**Depends on**: Story 3.2 (repository.list_jobs()).

```python
@router.get("/api/jobs")
async def list_jobs(repo: ResearchRepository = Depends(get_repository)):
    jobs = await repo.list_jobs(limit=50)
    return [j.model_dump(mode='json') for j in jobs]
```
