# Story 13.1: Error Boundary & Defensive Result Guards

## Status
Done

## Story
As a user running a research pipeline, I want the UI to show a helpful error message instead of a white screen when a rendering error occurs, so that I can retry or reload without losing context.

## Context
During a live research run, an OpenRouter 402 (insufficient credits) error caused the summary generation to fail. The backend recovered with a fallback, but the frontend crashed to a white screen — no error message, no retry button, no indication of what happened. After a manual page reload, the persisted results loaded correctly, confirming the data was fine but the live rendering transition was not protected.

Root causes:
1. No React Error Boundary anywhere in the component tree
2. Components like `VendorRankings` spread `rankings` (`[...rankings]`) before null-checking, crashing on undefined
3. `PriorityWeights` calls `.map()` on `requirements` without a guard

## Acceptance Criteria
- [x] An `ErrorBoundary` component exists that catches rendering errors and displays a fallback UI with a "Try Again" button
- [x] The results section in `ResearchPage` is wrapped with `ErrorBoundary` so crashes in any result component show the fallback instead of a white screen
- [x] `VendorRankings` guards against undefined/null `rankings` before spreading
- [x] `PriorityWeights` guards against undefined `requirements` before iterating
- [x] TypeScript type-check passes with no errors
- [x] ESLint passes with no errors
