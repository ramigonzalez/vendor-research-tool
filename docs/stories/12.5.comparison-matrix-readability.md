# Story 12.5: Comparison Matrix Readability

## Description
Fix 6 UX issues in the Comparison Matrix: broken priority grouping, cluttered requirement cells, confusing cell color encoding, invisible high-confidence indicator, raw decimal confidence values, and unlabeled vendor header scores.

## Acceptance Criteria
- [x] A: Priority section headers interleave correctly with their requirement rows
- [x] B: Requirement cells use two-line layout (ID label + description), no redundant priority badge
- [x] C: Score cells use neutral background with colored score bar + confidence percentage
- [x] D: Confidence indicator uses colored percentage text (green/amber/red)
- [x] E: Confidence displayed as percentage (66%) not decimal (0.66)
- [x] F: Vendor header score labeled as "64 / 100" not bare "(64.0)"
- [x] Legend updated to match new cell design
- [x] Priority Weights relocated to collapsible Ranking Breakdown card

## Files
- `src/components/organisms/ComparisonMatrix.tsx` — Fixes A, B, F, legend
- `src/components/atoms/ScoreCell.tsx` — Fixes C, D, E
- `src/components/molecules/PriorityWeights.tsx` — Collapsible card pattern
