# Story 12.5: Comparison Matrix Readability

## Description
Fix 6 UX issues in the Comparison Matrix plus add end-to-end status tracking for vendorГ—requirement pairs to distinguish "no evidence" from "search error."

## Acceptance Criteria
- [x] A: Priority section headers interleave correctly with their requirement rows
- [x] B: Requirement cells use two-line layout (ID label + description), no redundant priority badge
- [x] C: Score cells use neutral background with colored score bar + confidence percentage
- [x] D: Confidence indicator uses colored percentage text (green/amber/red) instead of border-left
- [x] E: Confidence displayed as percentage (66%) not decimal (0.66)
- [x] F: Vendor header score labeled as "64 / 100" not bare "(64.0)"
- [x] G: ScoreResult has `status` ("ok" | "degraded" | "error") + `status_detail` fields
- [x] H: Search errors tracked in pipeline state and propagated to ScoreResult
- [x] I: Frontend shows "No evidence" for degraded, warning icon + "Error" for errors
- [x] Legend updated to match new cell design (score bars, confidence colors, error states)
