## Story
**As a** research pipeline,
**I want** a single `get_llm()` factory that returns the correct `BaseChatModel` based on settings,
**so that** LLM construction logic is centralized and testable.

## Acceptance Criteria
1. `get_llm() -> BaseChatModel` added to `app/config.py`.
2. `openrouter` (default) returns `ChatOpenAI(model=settings.LLM_MODEL, temperature=settings.LLM_TEMPERATURE, base_url="https://openrouter.ai/api/v1", api_key=SecretStr(settings.OPENROUTER_API_KEY))`. Raises `ValueError` if `OPENROUTER_API_KEY` is empty.
3. `anthropic` returns `ChatAnthropic(model=settings.LLM_MODEL, temperature=settings.LLM_TEMPERATURE)`. Raises `ValueError` if `ANTHROPIC_API_KEY` is empty.
4. `openai` returns `ChatOpenAI(model=settings.LLM_MODEL, temperature=settings.LLM_TEMPERATURE, api_key=SecretStr(settings.OPENAI_API_KEY))`. Raises `ValueError` if `OPENAI_API_KEY` is empty.
5. Unknown provider raises `ValueError` listing valid providers.
6. Imports for `ChatAnthropic` / `ChatOpenAI` are **lazy** (inside function body) so neither package is required unless selected.
7. `langchain-openai>=0.3.0` added to `requirements.txt`.
8. New `tests/test_llm_factory.py` with 9 tests covering:
   - Default (openrouter) returns `ChatOpenAI` with correct `base_url`
   - Anthropic provider returns `ChatAnthropic`
   - OpenAI provider returns `ChatOpenAI` without custom `base_url`
   - Missing key for each provider raises `ValueError`
   - Unknown provider raises `ValueError`
   - Multiple keys set but only active provider's key is checked
   - Provider name is case-insensitive

## Dev Notes
**Depends on**: Story 6.1 (Settings fields)
**Tech Stack**: Python, langchain-anthropic, langchain-openai, pydantic `SecretStr`

**Key implementation details**:
- `ChatOpenAI` expects `api_key` as `SecretStr`, not `str` — wrap with `SecretStr(settings.OPENROUTER_API_KEY)`
- Provider name is lowercased before comparison: `settings.LLM_PROVIDER.lower()`
- The function has no return type annotation — pyright infers it correctly from branches

**Test pattern**:
```python
def _patch_settings(**overrides):
    """Patch settings with a FakeSettings object."""
    class FakeSettings: pass
    s = FakeSettings()
    for k, v in {**defaults, **overrides}.items():
        setattr(s, k, v)
    return patch("app.config.settings", s)
```
