## Story
**As a** frontend,
**I want** to retrieve cached research results by job ID,
**so that** completed results can be re-displayed without re-running the pipeline.

## Acceptance Criteria
1. `GET /api/research/{job_id}` returns JSON response.
2. Completed job: returns full `ResearchResults` with status 200.
3. Running job: returns `{"status": "running", "progress_pct": N}` with status 200.
4. Failed job: returns `{"status": "failed", "error": "<message>"}` with status 200.
5. Unknown job_id: returns 404 with `{"detail": "Job not found"}`.
6. Response for completed job includes: `vendors`, `requirements`, `matrix`, `rankings`, `summary`.
7. Matrix structure: `{"LangSmith": {"R1": {score, confidence, justification, ...}, ...}, ...}`.
8. Response time < 200ms for completed job (single SQLite read from repository).

## Tasks / Subtasks

## Dev Notes
**Depends on**: Story 3.2 (repository.get_results()), Story 2.1 (ResearchResults model).

**Route implementation**:
```python
from fastapi import HTTPException

@router.get("/api/research/{job_id}")
async def get_research_results(
    job_id: str,
    repo: ResearchRepository = Depends(get_repository)
):
    job = await repo.get_job(job_id)
    if job is None:
        raise HTTPException(status_code=404, detail="Job not found")

    if job.status == JobStatus.running:
        return {"status": "running", "progress_pct": job.progress_pct}

    if job.status == JobStatus.failed:
        return {"status": "failed", "error": job.progress_message}

    # completed
    results = await repo.get_results(job_id)
    if results is None:
        raise HTTPException(status_code=500, detail="Results not found for completed job")
    return results.model_dump(mode='json')
```

**matrix serialization**: `dict[str, dict[str, ScoreResult]]` serializes cleanly with Pydantic v2 `model_dump(mode='json')`. ScoreResult contains nested Evidence list.
