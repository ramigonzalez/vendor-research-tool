## Story
**As a** developer deploying the tool,
**I want** LLM provider, model, and temperature configurable via `.env` variables,
**so that** I can switch providers without editing source code.

## Acceptance Criteria
1. `app/config.py` `Settings` class gains new fields with defaults:
   - `LLM_PROVIDER: str = "openrouter"` (accepted: `anthropic`, `openrouter`, `openai`)
   - `LLM_MODEL: str = "meta-llama/llama-3.3-70b-instruct:free"` (default free model)
   - `LLM_TEMPERATURE: float = 0.0`
   - `OPENROUTER_API_KEY: str = ""` (required when provider=openrouter)
   - `OPENAI_API_KEY: str = ""` (required when provider=openai)
2. `ANTHROPIC_API_KEY` changes from required (`str`) to optional (`str = ""`). Validated at runtime only when provider=anthropic.
3. `.env.example` updated with all new variables documented, showing the multi-key setup pattern.
4. All API key fields default to `""`. Only the active provider's key is validated at runtime.

**Key implementation details**:

- Provider validation happens at runtime in `get_llm()` (Story 6.2), not in Settings
- The `model_config = {"env_file": ".env"}` remains unchanged — pydantic-settings loads all fields from `.env` automatically
