## Manual Test Procedure

### Prerequisites
1. Sign up at [openrouter.ai](https://openrouter.ai) and get a free API key
2. Set in `.env`:
   ```env
   LLM_PROVIDER=openrouter
   LLM_MODEL=meta-llama/llama-3.3-70b-instruct:free
   LLM_TEMPERATURE=0
   OPENROUTER_API_KEY=sk-or-your-key
   TAVILY_API_KEY=tvly-your-key
   ```

### Step 1: Run Integration Tests
```bash
OPENROUTER_API_KEY=sk-or-your-key pytest tests/test_openrouter_integration.py -v
```
Expected: 2 tests pass (basic completion + JSON output).

### Step 2: Start Server
```bash
./run.sh --skip-checks
```
Expected: Server starts, log shows `LLM: openrouter / meta-llama/llama-3.3-70b-instruct:free`.

### Step 3: Run Full Pipeline
1. Open `http://localhost:8000` in browser
2. Click "Run Research"
3. Observe progress updates via SSE
4. Wait for completion (~3–5 minutes with free model)
5. Verify: comparison matrix renders with scores for all 4 vendors x 6 requirements

### Step 4: Verify Results Quality
- Scores should be in reasonable ranges (not all 0s or all 10s)
- Evidence should contain real URLs and claims
- Executive summary should be coherent text

## Known Model Compatibility

| Model | JSON Reliability | Recommendation |
|-------|-----------------|----------------|
| `meta-llama/llama-3.3-70b-instruct:free` | Good | Recommended — best JSON output among free models |
| `google/gemma-2-9b-it:free` | Fair | Smaller model, may produce malformed JSON occasionally |
| `mistralai/mistral-7b-instruct:free` | Poor | Struggles with structured output, not recommended |

**Notes**:
- Free models have rate limits on OpenRouter (varies by model and demand)
- JSON parsing failures are handled gracefully — fallback queries and default assessments are used
- For production quality results, switch to `LLM_PROVIDER=anthropic` with `LLM_MODEL=claude-sonnet-4-5`
