# Story 12.6: End-to-End Score Status Tracking

## Description
The Comparison Matrix displayed identical cells (score 2.4, confidence 0%) for both "search errored due to rate limit" and "searched but found nothing relevant." Users could not distinguish between a failed search and a genuinely unresearched vendor├Чrequirement pair. This story adds a `status` + `status_detail` field to `ScoreResult` that propagates from the backend pipeline through to the frontend UI.

## Problem
The backend scoring formula produces a default score of **2.4** when no evidence exists:
```
Score = 0.4├Чunknown(2) + 0.3├ЧnoEvidence(0) + 0.2├Чunknown(3) + 0.1├ЧnoLimitations(10) = 2.4
```

Four different failure modes all produced this same output:

| Scenario | Root Cause | Previous Display |
|----------|-----------|-----------------|
| Tavily rate limit / timeout | `except Exception` тЖТ silent `[]` | 2.4 / 0% |
| Search returned zero results | No matching web content | 2.4 / 0% |
| Evidence found but all filtered | `relevance < 0.3` threshold | 2.4 / 0% |
| LLM assessment crashed | Model API error | 2.4 / 0% |

## Solution

### Backend: `ScoreResult` model (`app/models.py`)
Added two fields with backward-compatible defaults:
- `status: Literal["ok", "degraded", "error"]` тАФ defaults to `"ok"`
- `status_detail: str | None` тАФ human-readable context, defaults to `None`

### Backend: Search error tracking (`app/graph/nodes.py`, `app/graph/state.py`)
- Added `search_errors: dict[str, str]` to `ResearchState` тАФ maps `vendor:req_id` to error message
- `execute_searches` now captures exception messages per pair into `search_errors`
- `compute_scores` reads `search_errors` + evidence + assessment to determine status:

| Condition | Status | Example `status_detail` |
|-----------|--------|------------------------|
| Search threw exception | `error` | `"Search failed: Tavily rate limit exceeded"` |
| No evidence + default assessment | `degraded` | `"No evidence found for this requirement"` |
| Evidence found, all below relevance | `degraded` | `"3 sources found, all below relevance threshold"` |
| Low confidence + unknown capability | `degraded` | `"Insufficient evidence for reliable assessment"` |
| Normal scoring path | `ok` | `null` |

### Frontend: Types (`src/lib/types.ts`)
- Added `ScoreStatus = 'ok' | 'degraded' | 'error'`
- Added `status` and `status_detail` fields to `ScoreResult` interface

### Frontend: ScoreCell (`src/components/atoms/ScoreCell.tsx`)
Three visual states based on `status`:
- **`ok`** тАФ Score bar + colored confidence percentage (normal display)
- **`degraded`** тАФ *"No evidence"* in muted italic text
- **`error`** тАФ Warning icon + *"Error"* in red, with `status_detail` as hover tooltip

Backward compatible: if `status` is undefined (old data), falls back to `confidence > 0` check.

### Frontend: Legend (`src/components/organisms/ComparisonMatrix.tsx`)
Updated confidence legend to explain all three states:
- Colored percentage text for high/medium/low confidence
- *"No evidence = searched, nothing found"*
- *"тЪа Error = search failed"*
