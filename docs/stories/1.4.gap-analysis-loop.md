## Story
**As a** research pipeline,
**I want** to detect evidence gaps after initial research and run a targeted second iteration to fill them,
**so that** sparse-evidence pairs get a second chance with refined queries before scoring.

## Acceptance Criteria
1. `check_evidence_sufficiency(evidence: list[Evidence]) -> bool` function with deterministic thresholds: ≥2 sources AND ≥1 authoritative (source_type in {official_docs, github}) AND ≥1 with relevance ≥ 0.5.
2. `find_evidence_gaps(state: ResearchState) -> list[tuple[str, str]]` returns (vendor, req_id) pairs failing sufficiency.
3. `diagnose_gap(evidence: list[Evidence]) -> GapType` classifies gap: `no_evidence`, `no_authoritative_source`, `low_relevance`, `insufficient_count`.
4. Gap type drives refined query strategy:
   - `no_evidence` → try alternative terminology (e.g., "monitoring" instead of "observability")
   - `no_authoritative_source` → add `site:github.com` or `site:docs.<vendor>.com`
   - `low_relevance` → narrow with more specific terms from requirement description
   - `insufficient_count` → add community/changelog queries
5. LangGraph conditional edge: `should_continue_research(state) -> str` returns "continue" if gaps exist AND `state.iteration < 2`, else "proceed_to_scoring".
6. Maximum 2 research iterations enforced.
7. `state.gaps` stores gap metadata: `{(vendor, req_id): GapType}` for UI transparency.
8. `state.iteration` incremented after each research round.

## Dev Notes
**Depends on**: Story 1.3 (evidence extracted), Story 2.1 (GapType enum in models).

**Evidence sufficiency thresholds** (from final planning doc):
```python
def check_evidence_sufficiency(evidence: list[Evidence]) -> bool:
    if len(evidence) < 2:
        return False
    has_authoritative = any(
        e.source_type in (SourceType.official_docs, SourceType.github)
        for e in evidence
    )
    if not has_authoritative:
        return False
    has_relevant = any(e.relevance >= 0.5 for e in evidence)
    return has_relevant
```

**Gap diagnosis logic**:
```python
def diagnose_gap(evidence: list[Evidence]) -> GapType:
    if not evidence:
        return GapType.no_evidence
    if not any(e.source_type in (SourceType.official_docs, SourceType.github) for e in evidence):
        return GapType.no_authoritative_source
    if not any(e.relevance >= 0.5 for e in evidence):
        return GapType.low_relevance
    return GapType.insufficient_count
```

**Conditional edge**: LangGraph routes based on string return value:
```python
def should_continue_research(state: ResearchState) -> str:
    gaps = find_evidence_gaps(state)
    if gaps and state["iteration"] < 2:
        state["gaps"] = {pair: diagnose_gap(state["evidence"].get(pair, [])) for pair in gaps}
        return "continue"
    return "proceed"
```

**Refined query generation** for gap-filling is a simplified version of Story 1.1 — may reuse `generate_queries` node with gap context appended to prompt.

**State update**: `state["iteration"] += 1` before looping back to generate_queries.
