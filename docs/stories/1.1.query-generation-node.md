## Story
**As a** research pipeline,
**I want** a LangGraph node that generates 2 targeted search queries per vendor×requirement pair using an LLM,
**so that** each pair is researched from two complementary angles to maximize evidence coverage.

## Acceptance Criteria
1. LangGraph node function `generate_queries(state: ResearchState) -> ResearchState` in `app/graph/nodes.py`.
2. Node calls LLM once per vendor×requirement pair with a structured prompt requesting exactly 2 queries.
3. Query 1 targets official sources: vendor docs, GitHub repository, official blog.
4. Query 2 targets external sources: comparisons, community discussions, blog posts, case studies.
5. LLM output parsed into `{"query_1": "...", "query_2": "..."}` structured format.
6. Fallback queries generated using template strings if LLM output is invalid/unparseable: `f"{vendor} {requirement} official documentation site:docs.{vendor.lower()}.com"` and `f"{vendor} {requirement} comparison review 2024"`.
7. All 24 query pairs (4 vendors × 6 requirements) stored in `state.queries` as `dict[(vendor, req_id), tuple[str, str]]`.
8. Node is async (`async def generate_queries`).
9. LLM calls for all 24 pairs run concurrently (asyncio.gather).

## Dev Notes
**Depends on**: Story 2.1 (ResearchState TypedDict), Story 5.1 (config with VENDORS, REQUIREMENTS).

**LLM client initialization** (LangChain Anthropic):
```python
from langchain_anthropic import ChatAnthropic
from langchain_core.output_parsers import JsonOutputParser

llm = ChatAnthropic(model="claude-sonnet-4-5", temperature=0)
```

**Structured output approach** — use `with_structured_output` or `JsonOutputParser`:
```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages([
    ("system", QUERY_GENERATION_SYSTEM_PROMPT),
    ("user", "Vendor: {vendor}\nRequirement: {requirement}\nPriority: {priority}")
])
chain = prompt | llm | JsonOutputParser()
```

**Query generation prompt** (from planning docs):
```
Generate 2 search queries for researching {vendor}'s {requirement} capability.
Query 1: Target official documentation, GitHub, or product pages.
Query 2: Target comparisons, community discussions, or third-party reviews.
Return JSON: {"query_1": "...", "query_2": "..."}
Keep queries under 15 words. Be specific. Include year 2024 or 2025 in at least one query.
```

**Examples from planning docs**:
- LangSmith × Framework-agnostic tracing:
  - Q1: "LangSmith tracing SDK framework agnostic OpenTelemetry support site:smith.langchain.com"
  - Q2: "LangSmith vs Langfuse tracing non-LangChain frameworks comparison 2024"

**Fallback validation**: if JSON parse fails or keys missing, use template strings (not empty strings).

**`asyncio.gather`**: all 24 LLM calls in parallel — reduces generation time significantly.
