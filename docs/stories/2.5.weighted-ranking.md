## Story
**As a** results consumer,
**I want** vendors ranked by a weighted score that combines requirements, priorities, and confidence,
**so that** the executive summary and UI display a clear, justifiable recommendation.

## Acceptance Criteria
1. `compute_vendor_rankings(scores: dict, requirements: list[Requirement]) -> list[VendorRanking]` in `app/scoring/engine.py`.
2. Formula: `weighted_score = Σ(score × priority_weight × confidence) / max_possible × 100`, normalized to 0–100. Priority weights: High=3.0, Medium=2.0, Low=1.0 (from `config.PRIORITY_WEIGHTS`).
3. Rankings sorted descending by weighted_score.
4. Each `VendorRanking` includes: `vendor: str`, `overall_score: float` (2 decimal places), `rank: int` (1-based).
5. All 4 vendors always returned (no filtering).
6. Tiebreaker for equal scores: alphabetical vendor name (stable, deterministic).
7. LangGraph node `compute_rankings(state: ResearchState) -> ResearchState` calls this function.
8. Rankings stored in `state.rankings` and persisted to SQLite via `repository.save_final_results()`.
9. Function is pure.

**Ranking computation** (from Doc 2 `compute_overall_rankings()`):
```python
from app.config import PRIORITY_WEIGHTS

def compute_vendor_rankings(
    scores: dict[str, dict[str, ScoreResult]],  # {vendor: {req_id: ScoreResult}}
    requirements: list[Requirement]
) -> list[VendorRanking]:
    """
    Priority-based weighted ranking.
    Weight comes from priority category: High=3.0, Medium=2.0, Low=1.0
    Formula: Σ(score × priority_weight × confidence) / max_possible × 100
    """
    # Maximum possible score (for normalization)
    max_possible = sum(
        10.0 * PRIORITY_WEIGHTS[req.priority] * 1.0  # score=10, confidence=1
        for req in requirements
    )

    rankings = []
    for vendor, req_scores in scores.items():
        weighted_sum = 0.0
        total_confidence = 0.0
        for req in requirements:
            req_id = str(req.id)
            if req_id in req_scores:
                result = req_scores[req_id]
                weight = PRIORITY_WEIGHTS[req.priority]
                weighted_sum += result.score * weight * result.confidence
                total_confidence += result.confidence

        normalized_score = (weighted_sum / max_possible) * 100 if max_possible > 0 else 0
        avg_confidence = total_confidence / len(requirements) if requirements else 0
        rankings.append({
            "vendor": vendor,
            "weighted_score": round(normalized_score, 2),
            "avg_confidence": round(avg_confidence, 2),
        })

    sorted_rankings = sorted(rankings, key=lambda x: (-x["weighted_score"], x["vendor"]))
    return [
        VendorRanking(vendor=r["vendor"], overall_score=r["weighted_score"], rank=i+1)
        for i, r in enumerate(sorted_rankings)
    ]
```

**Note on scaling**: Normalized to 0–100 scale (matching Doc 1 §2.5 Layer 3 and Doc 2 implementation).

**Priority weights** (from `config.PRIORITY_WEIGHTS`, derived from priority category):
- High = 3.0 (R1, R2)
- Medium = 2.0 (R3, R4)
- Low = 1.0 (R5, R6)
