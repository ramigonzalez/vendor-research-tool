# Story 11.1: UX Overhaul — Comparison Matrix Readability & Component Polish

**Epic:** 11 — UX Overhaul: Matrix Readability & Component Polish
**Priority:** CRITICAL → MEDIUM (phased)
**Executor:** @dev
**Quality Gate:** @ux-design-expert

## User Story

As a product evaluator reviewing vendor comparison results, I want a visually polished and readable UI with proper color semantics, rendered markdown, and complete requirement context, so that I can quickly understand and trust the research output.

## Acceptance Criteria

### Phase A — CRITICAL
- [x] Design tokens `--color-bg-table-header` and `--color-bg-table-section` added to globals.css (light + dark)
- [x] Markdown prose utility class `.markdown-prose` added to globals.css
- [x] ComparisonMatrix headers use `bg-bg-table-header` instead of `bg-text-primary`
- [x] Priority section rows use per-priority tinted backgrounds with colored left borders (not hardcoded dark)
- [x] Each requirement row shows `[Priority Badge] req.id description`
- [x] ConfidenceBreakdown uses per-type colors (teal/blue/purple/copper) not per-value (green/orange/red)
- [x] ScoringMethodology Source Count bar matches new confidence colors

### Phase B — HIGH
- [x] react-markdown installed as dependency
- [x] ExecutiveSummary renders markdown via ReactMarkdown with Source Serif 4 font
- [x] ExecutiveSummary has Rendered/Source view toggle
- [x] PriorityWeights per-vendor breakdown uses larger text, card styling, and shows req descriptions
- [x] ResearchTimeline auto-collapses 2s after completion with smooth CSS transition

### Phase C — MEDIUM
- [x] Rankings in ExecutiveSummary show rank badge, score bar, and #1 highlight
