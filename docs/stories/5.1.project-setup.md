## Story
**As a** developer setting up the project,
**I want** a complete project scaffold with pinned dependencies and environment configuration,
**so that** the tool runs with `git clone → cp .env.example .env → pip install -r requirements.txt → uvicorn app.main:app`.

## Acceptance Criteria
1. `requirements.txt` with pinned versions for all dependencies.
2. `.env.example` with `ANTHROPIC_API_KEY=sk-ant-xxx` and `TAVILY_API_KEY=tvly-xxx`.
3. `.gitignore` includes: `research.db`, `.env`, `__pycache__/`, `*.pyc`, `.pytest_cache/`, `venv/`.
4. `app/config.py` uses `pydantic-settings` `BaseSettings` to load API keys from env.
5. `app/config.py` defines `VENDORS: list[str]` = ["LangSmith", "Langfuse", "Braintrust", "PostHog"].
6. `app/config.py` defines `REQUIREMENTS: list[Requirement]` = 6 requirements with priority levels (High, Medium, Low). Priority weights (High=3.0, Medium=2.0, Low=1.0) are defined as a constant dict, not per-requirement.
7. All `__init__.py` files created: `app/`, `app/graph/`, `app/scoring/`, `app/prompts/`.
8. `tests/conftest.py` with pytest fixtures for mock repository.
9. `tests/__init__.py` created.
10. `uvicorn app.main:app --reload` starts without import errors.

**app/config.py**:
```python
from pydantic_settings import BaseSettings
from app.models import Requirement

class Settings(BaseSettings):
    ANTHROPIC_API_KEY: str
    TAVILY_API_KEY: str

    model_config = {"env_file": ".env"}

settings = Settings()

VENDORS = ["LangSmith", "Langfuse", "Braintrust", "PostHog"]

PRIORITY_WEIGHTS = {
    "high": 3.0,
    "medium": 2.0,
    "low": 1.0,
}

REQUIREMENTS = [
    Requirement(id="R1", description="Framework-agnostic tracing", priority="high"),
    Requirement(id="R2", description="Self-hosting support", priority="high"),
    Requirement(id="R3", description="Evaluation framework", priority="medium"),
    Requirement(id="R4", description="OpenTelemetry integration", priority="medium"),
    Requirement(id="R5", description="Custom metrics", priority="low"),
    Requirement(id="R6", description="Cost transparency", priority="low"),
]
```

**Priority weights**: Derived from priority category (High=3.0, Medium=2.0, Low=1.0), as specified in initial documentation (Doc 1 §2.5 Layer 3, Doc 2 `compute_overall_rankings()`).
