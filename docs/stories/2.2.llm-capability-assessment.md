## Story
**As a** scoring engine,
**I want** an LLM to produce a structured `LLMAssessment` for each vendor×requirement pair from collected evidence,
**so that** the deterministic scoring engine has structured, validated input to compute scores from.

## Acceptance Criteria
1. LangGraph node `assess_capabilities(state: ResearchState) -> ResearchState` in `app/graph/nodes.py`.
2. LLM receives per-pair context: vendor, requirement description, priority, and list of evidence (claim + source_url + source_type + relevance + supports_requirement).
3. LLM outputs `LLMAssessment` via structured output: `capability_level`, `capability_details`, `maturity`, `limitations: list[str]`, `supports_requirement: bool`.
4. LLM prompt includes conservative guidance: use `unknown` when evidence is ambiguous, not `partial` or `minimal`.
5. Evidence passed to LLM includes only evidence where `relevance >= 0.3` (filter noise).
6. Assessments stored in `state.assessments` as `dict[tuple[str, str], LLMAssessment]`.
7. If LLM fails or returns invalid output: default to `LLMAssessment(capability_level=unknown, maturity=unknown, capability_details="Assessment failed", limitations=[], supports_requirement=False)`.
8. All 24 assessments run concurrently.

**Assessment prompt** (from planning docs):
```
You are assessing a vendor's capability for a specific requirement based ONLY on the provided evidence.

Vendor: {vendor}
Requirement: {requirement} (Priority: {priority})

Evidence (sorted by relevance):
{formatted_evidence}

Assess the vendor's capability. RULES:
1. Base your assessment ONLY on the provided evidence, not your training knowledge.
2. Use "unknown" if evidence is insufficient or ambiguous — do NOT guess.
3. capability_level choices: full (clearly and completely supported), partial (some support but gaps), minimal (barely addressed), none (explicitly not supported), unknown (insufficient evidence).
4. maturity choices: ga (generally available, production-ready), beta (available but not production), experimental (in development), planned (on roadmap), unknown.
5. List specific limitations as quoted claims from sources.

Return JSON matching LLMAssessment schema.
```

**Structured output with Pydantic**:
```python
from langchain_anthropic import ChatAnthropic

llm = ChatAnthropic(model="claude-sonnet-4-5", temperature=0)
structured_llm = llm.with_structured_output(LLMAssessment)
```

**Evidence formatting for LLM** (show claim, source_type, relevance):
```python
def format_evidence_for_llm(evidence: list[Evidence]) -> str:
    lines = []
    for i, e in enumerate(sorted(evidence, key=lambda x: -x.relevance)):
        lines.append(f"{i+1}. [{e.source_type.value}] (relevance: {e.relevance:.2f})")
        lines.append(f"   Claim: {e.claim[:500]}")
        lines.append(f"   Source: {e.source_name} ({e.source_url[:80]})")
        lines.append(f"   Supports: {e.supports_requirement}")
    return "\n".join(lines)
```

**Conservative default**: On any exception, return `LLMAssessment(capability_level=CapabilityLevel.unknown, ...)` — never let assessment failure crash the pipeline.
