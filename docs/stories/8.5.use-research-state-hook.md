## Story
**As a** developer,
**I want** a centralized state management hook for the research pipeline,
**so that** all UI components stay in sync with the backend's SSE events through a single source of truth.

## Acceptance Criteria
1. [x] State machine phases: `idle -> planning -> searching -> analyzing -> scoring -> ranking -> writing -> complete | error`.
2. [x] Tracks: currentPhase, stepStatuses, sources[], queries[], evidenceCounts, scores, rankings, elapsedTime, sourceCount.
3. [x] SSE client: handles connection, abort, parsing via fetch ReadableStream.
4. [x] All UI components consume this hook.
5. [x] State resets on new research run.
6. [x] Activity counter uses `useRef` (not module-level mutable state).
