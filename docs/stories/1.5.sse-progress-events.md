## Story
**As a** frontend user,
**I want** to receive real-time progress events while the research pipeline runs,
**so that** I see meaningful feedback with phase labels and percentages instead of a frozen screen.

## Acceptance Criteria
1. Pipeline emits `{"type": "started", "job_id": "<uuid>"}` at start.
2. Pipeline emits `{"type": "progress", "phase": "<phase>", "pct": <int>, "message": "<str>"}` at milestones.
3. Phase values and percentage ranges: `research` (5–50%), `scoring` (50–80%), `synthesis` (80–95%).
4. Pipeline emits `{"type": "completed", "results": {...ResearchResults...}}` on success.
5. Pipeline emits `{"type": "error", "message": "<str>"}` on failure.
6. Events formatted as SSE: each line is `data: {json_string}\n\n`.
7. Progress queue (`asyncio.Queue`) passed through `ResearchState` enables pipeline nodes to enqueue events.
8. FastAPI `StreamingResponse` dequeues and yields events as SSE lines.
9. Progress percentages are meaningful (not just 0% → 100%): at least 5 distinct progress updates emitted during a full run.

## Dev Notes
**Depends on**: Story 2.1 (ResearchState with progress_queue field), Story 3.3 (FastAPI SSE endpoint).

**Progress queue pattern**:
```python
# In ResearchState TypedDict:
progress_queue: asyncio.Queue  # Queue[dict]

# Helper function:
async def emit_progress(state: ResearchState, phase: str, pct: int, message: str):
    await state["progress_queue"].put({
        "type": "progress",
        "phase": phase,
        "pct": pct,
        "message": message
    })
```

**SSE generator** (called from FastAPI endpoint):
```python
async def run_and_stream(job_id: str, repo: ResearchRepository):
    queue = asyncio.Queue()
    state = build_initial_state(job_id, queue)

    # Yield started event
    yield f"data: {json.dumps({'type': 'started', 'job_id': job_id})}\n\n"

    # Run pipeline in background and consume queue
    pipeline_task = asyncio.create_task(run_pipeline(state, repo))

    while not pipeline_task.done() or not queue.empty():
        try:
            event = await asyncio.wait_for(queue.get(), timeout=0.1)
            yield f"data: {json.dumps(event)}\n\n"
        except asyncio.TimeoutError:
            continue

    # Emit final completed or error
    if pipeline_task.exception():
        yield f"data: {json.dumps({'type': 'error', 'message': str(pipeline_task.exception())})}\n\n"
    else:
        results = await repo.get_results(job_id)
        yield f"data: {json.dumps({'type': 'completed', 'results': results.model_dump()})}\n\n"
```

**SSE format**: `data: {json}\n\n` — double newline is required by SSE spec.

**Frontend consumption**: uses `fetch` with `ReadableStream` + `TextDecoder`, not `EventSource` (EventSource doesn't support POST with custom headers).
