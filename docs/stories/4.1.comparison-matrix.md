## Story
**As a** researcher/evaluator,
**I want** a comparison matrix showing all vendor scores per requirement,
**so that** I can quickly assess vendor capabilities at a glance.

## Acceptance Criteria
1. Matrix renders with vendors as columns, requirements as rows in `static/index.html`.
2. Each cell shows numerical score (0–10, one decimal place, e.g., "7.3").
3. Cell background color: score ≥7 → green (#2ecc71), 4–6.9 → amber (#f39c12), <4 → red (#e74c3c). Text color always white.
4. Requirements grouped: "High Priority" rows first, "Medium Priority", "Low Priority" — separated by section header rows.
5. Vendor name + overall rank displayed in column header (e.g., "LangSmith\n#1 (8.4)").
6. Missing/null scores display as "—" in neutral style.
7. Matrix container horizontally scrollable on screens < 768px.
8. Matrix renders correctly with results from `GET /api/research/{job_id}`.

## Dev Notes
**Depends on**: Story 3.4 (GET results endpoint providing matrix data).

**Results matrix structure** (from API):
```json
{
  "matrix": {
    "LangSmith": {
      "R1": {"score": 8.5, "confidence": 0.85, "justification": "...", "evidence": [...], "limitations": [...]},
      "R2": {...}
    },
    "Langfuse": {...}
  },
  "rankings": [{"vendor": "LangSmith", "overall_score": 8.1, "rank": 1}],
  "requirements": [{"id": "R1", "description": "Framework-agnostic tracing", "priority": "high", "weight": 0.25}]
}
```

**Score color function**:
```javascript
function getScoreColor(score) {
  if (score === null) return '#6c757d';  // grey
  if (score >= 7) return '#2ecc71';      // green
  if (score >= 4) return '#f39c12';      // amber
  return '#e74c3c';                      // red
}
```

**Priority grouping**: sort requirements by priority order (high → medium → low). Use priority section header `<tr class="priority-header"><td colspan="N">High Priority</td></tr>`.

**Responsive table**: wrap table in `<div style="overflow-x: auto">`.
