## Story
**As a** user viewing research results,
**I want** to see full requirement descriptions instead of "R1", "R2" codes,
**so that** I can understand what each requirement means without referring to external documentation.

## Acceptance Criteria
1. [x] `get_results()` in `repository.py` imports `REQUIREMENTS` from `app.config`.
2. [x] Builds lookup dict `{req.id: req for req in REQUIREMENTS}` to resolve descriptions and priorities.
3. [x] Falls back to `description=rid, priority=medium` for unknown requirement IDs.
4. [x] Unit test verifying description resolution from config.
5. [x] Unit test verifying fallback for unknown requirement ID.
