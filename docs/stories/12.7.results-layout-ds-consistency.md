# Story 12.7: Results Layout & Design System Consistency

## Description
After completing Stories 12.1–12.6 (theme, home page, collapsible audit, executive summary, comparison matrix, score status), a UX audit identified 7 remaining issues. This story addresses all 7 in one pass to bring the results section into full design system consistency.

## Issues Addressed

1. **Rankings buried inside Executive Summary** — Extract to standalone `VendorRankings` card, display before matrix
2. **Comparison Matrix missing container card** — Wrap in `bg-bg-secondary` card with border
3. **Nested collapse chaos in Research Analysis** — Remove inner collapse toggles from ResearchTimeline and ActivityFeed
4. **Executive Summary text not filling card** — Remove `max-w-prose` constraint
5. **JobHistory styled inconsistently** — Restyle toggle to match disclosure pattern
6. **Sub-component padding/margin violations** — Standardize to `p-4` in QueryHistory, SourcesVisited, ActivityFeed
7. **Timeline card-in-card nesting** — Remove card wrapper from ResearchTimeline (parent AuditView provides it)

## Acceptance Criteria
- [x] VendorRankings appears as standalone card between AuditView and ComparisonMatrix
- [x] ComparisonMatrix wrapped in card with visible hover on cells
- [x] No nested collapse toggles inside Research Analysis tabs
- [x] Executive Summary text fills card width
- [x] JobHistory toggle matches ScoringMethodology disclosure style
- [x] All tab panels use consistent `p-4` padding
- [x] No card-in-card nesting in timeline
- [x] Lint and typecheck pass with zero errors
