# Wave Log: Story 2.3 - Deterministic Score Computation

## Status: COMPLETE

## Date: 2026-02-24

## Changes

### app/scoring/engine.py
- Added `CAPABILITY_SCORES` lookup table mapping `CapabilityLevel` enum values to 0-10 scores
- Added `MATURITY_SCORES` lookup table mapping `MaturityLevel` enum values to 0-10 scores
- Added `compute_requirement_score()` pure function that computes a weighted 0-10 score from:
  - Capability level (40%)
  - Supporting evidence count, capped at 5 (30%)
  - Maturity level (20%)
  - Limitations penalty, clamped at 0 (10%)

### app/graph/nodes.py
- Added `compute_scores()` LangGraph node that iterates all vendor-requirement pairs
- Computes both `score` (via `compute_requirement_score`) and `confidence` (via `compute_confidence`)
- Assembles `ScoreResult` objects and returns `{"scores": ...}` state update
- Falls back to `_DEFAULT_ASSESSMENT` (from Story 2.2) when no assessment exists
- Emits progress event at 80%

### tests/test_scoring.py
- Added `TestComputeRequirementScore` class with 6 tests:
  - Perfect score (full/ga/5 evidence/0 limitations = 10.0)
  - None capability with unknown maturity = 1.6
  - Unknown capability with unknown maturity = 2.4
  - Limitations score clamped at 0 (not negative)
  - Evidence count capped at 5
  - Partial/beta/3 evidence/1 limitation = 6.8
- Added `TestCapabilityAndMaturityScores` class verifying all lookup table values and completeness
