# Wave Log: Story 2.2 — LLM Capability Assessment

## Status: Complete

## Summary
Implemented the `assess_capabilities` LangGraph node that takes structured evidence from Story 1.3 and uses an LLM to produce `LLMAssessment` objects for each vendor-requirement pair, evaluating capability level, maturity, limitations, and requirement support.

## Files Created
- `app/prompts/assessment.py` — System and user prompt templates for capability assessment

## Files Modified
- `app/graph/nodes.py` — Appended `_format_evidence_for_llm()`, `_DEFAULT_ASSESSMENT`, `_assess_single_pair()`, and `assess_capabilities()` node
- `tests/test_nodes.py` — Appended `TestAssessCapabilities` class with 5 tests

## Key Decisions
- Evidence with relevance < 0.3 is filtered out before sending to the LLM
- If no evidence passes the relevance filter, the default assessment is returned without calling the LLM
- Default assessment uses `CapabilityLevel.unknown` and `MaturityLevel.unknown` with "Assessment failed" details
- Evidence is sorted by descending relevance before formatting for the LLM
- Claims are truncated to 500 characters and source URLs to 80 characters in the formatted prompt
- LLM failures are logged as warnings and fall back to the default assessment (graceful degradation)
- All vendor-requirement pairs are assessed concurrently via `asyncio.gather`
