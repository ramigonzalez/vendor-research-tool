# Wave 3 — Story 1.4: Evidence Gap Analysis & Research Loop

## Status: COMPLETE

## Changes Made

### 1. `app/models.py` — Added `GapType` enum
- New enum with four gap types: `no_evidence`, `no_authoritative_source`, `low_relevance`, `insufficient_count`
- Placed after `Priority` enum, before `Evidence` class

### 2. `app/graph/nodes.py` — Appended gap analysis functions
- `check_evidence_sufficiency(evidence)` — validates minimum quality thresholds (>= 2 sources, at least one authoritative, at least one with relevance >= 0.5)
- `diagnose_gap(evidence)` — classifies evidence gap into a `GapType`
- `find_evidence_gaps(state)` — scans all vendor-requirement pairs for insufficient evidence
- `should_continue_research(state)` — sync conditional edge: returns `"continue"` or `"proceed"` based on iteration count and gaps
- `generate_refined_queries(vendor, requirement_desc, gap_type)` — produces 2 targeted queries per gap type
- `prepare_gap_filling(state)` — async node that assembles gap metadata, refined queries, and increments iteration

### 3. `tests/test_evidence.py` — 27 new tests
- `TestCheckEvidenceSufficiency` — 6 tests covering empty, single source, no authoritative, low relevance, and valid cases
- `TestDiagnoseGap` — 4 tests covering all `GapType` classifications
- `TestShouldContinueResearch` — 4 tests for iteration limits and gap detection
- `TestGenerateRefinedQueries` — 6 tests verifying query count and content per gap type
- `TestFindEvidenceGaps` — 3 tests for gap identification and empty results
- `TestPrepareGapFilling` — 4 tests for iteration increment, query generation, metadata, and no-gap case
