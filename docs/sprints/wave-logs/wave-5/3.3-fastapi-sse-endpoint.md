# Wave Log: Story 3.3 — POST /api/research with SSE Streaming

## Summary
Added the POST /api/research endpoint that launches the research pipeline and streams progress events via Server-Sent Events (SSE). Also added CORS middleware and the `build_initial_state` helper function.

## Files Modified
- `app/main.py` — Added CORS middleware, converted try/except router import to direct import
- `app/api/router.py` — Added POST /api/research SSE streaming endpoint with pipeline integration
- `app/graph/state.py` — Added `build_initial_state()` factory function
- `tests/test_api.py` — Added 5 tests for the SSE endpoint (stream type, started event, completed event, job creation, pipeline invocation)

## Changes Detail

### app/graph/state.py
- Added `build_initial_state(job_id, queue)` function that creates a fully initialized `ResearchState` with vendors and requirements from config.

### app/main.py
- Added `CORSMiddleware` allowing origins `http://localhost:8000` and `http://127.0.0.1:8000`
- Replaced try/except router import with direct import since the router module now exists

### app/api/router.py
- Added `POST /api/research` endpoint that:
  - Creates a new job in the repository with `running` status
  - Returns a `StreamingResponse` with `text/event-stream` media type
  - Emits a `started` event with the job ID
  - Launches the pipeline as an asyncio task
  - Streams progress events from the queue
  - Emits `completed` or `error` events based on pipeline outcome

### tests/test_api.py
- Added `TestRunResearch` class with 5 tests using `unittest.mock.patch` to mock `run_pipeline`
- Tests verify: SSE content type, started event structure, completed event, job persistence, and pipeline invocation
