## Key Decisions
1. **Concurrent execution**: All vendor x requirement LLM calls run concurrently via `asyncio.gather` for maximum throughput.
2. **Fallback strategy**: Two fallback queries generated when LLM response is unparseable or incomplete (missing `query_1` or `query_2`). Fallback query 1 targets official docs; fallback query 2 targets comparison reviews.
3. **Key format**: State queries dict uses `"{vendor}:{req_id}"` composite keys as specified.
4. **LangGraph node pattern**: Function returns partial state dict `{"queries": ...}` rather than full state.
5. **Helper function `_generate_query_pair`**: Extracted to enable concurrent execution and clean separation of per-pair logic.
