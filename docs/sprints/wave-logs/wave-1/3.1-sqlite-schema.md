## Key Decisions
- `create_db_and_tables()` accepts an optional `db_path` parameter to allow tests to use `:memory:` or temp files without monkeypatching
- WAL mode test uses a temp file (WAL is not supported on `:memory:` databases)
- Tests use a helper `create_db_and_tables_on_conn()` to apply schema on an already-open connection, since `:memory:` databases are per-connection
- SQL DDL statements stored as module-level constants for reuse in tests
