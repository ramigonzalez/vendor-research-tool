# Wave Log: Story 3.2 - Repository Pattern Implementation

## Date: 2026-02-24

## Summary
Implemented the Repository Pattern for the Vendor Research Tool, adding an abstract `ResearchRepository` ABC and a concrete `SQLiteResearchRepository` backed by aiosqlite. Replaced the placeholder mock with a full `MockResearchRepository` that extends the ABC for testing.

## Changes Made

### app/repository.py
- Added `ResearchRepository` ABC with 8 abstract methods: `create_job`, `update_job_status`, `save_evidence`, `save_score`, `get_job`, `get_results`, `list_jobs`, `save_final_results`
- Added `SQLiteResearchRepository(ResearchRepository)` with full SQLite implementation:
  - All DML uses parameterized `?` placeholders (no f-strings in SQL)
  - `save_score` uses INSERT OR REPLACE for upsert semantics
  - `save_evidence` bulk-inserts via loop
  - `save_final_results` updates job to completed with summary and rankings JSON
  - `get_results` joins scores, evidence, and job data to build `ResearchResults`
  - `limitations_json` serialized with json.dumps/json.loads
  - `supports_requirement` stored as int(bool), read as bool(int)
- Added `get_repository()` singleton accessor
- Preserved existing `create_db_and_tables()` and all table constants unchanged

### tests/conftest.py
- Replaced simple `MockResearchRepository` with full ABC-compliant implementation
- In-memory dicts for `_jobs`, `_evidence`, `_scores`, `_summaries`, `_rankings`
- All 8 abstract methods implemented
- `mock_repository` fixture preserved

### tests/test_repository.py
- Preserved all existing schema tests (table creation, columns, WAL, constraints, indexes, idempotency)
- Added 10 CRUD tests:
  - `test_create_job_and_get_job` - round-trip create/read
  - `test_get_job_returns_none_for_missing` - None for non-existent ID
  - `test_update_job_status` - status/progress update
  - `test_save_evidence_and_get_results` - evidence via get_results
  - `test_save_score_and_get_results` - score matrix entry
  - `test_save_final_results` - completed status, summary, rankings
  - `test_get_results_correct_matrix_structure` - multi-vendor/req matrix
  - `test_get_results_returns_none_for_missing_job` - None for missing
  - `test_list_jobs_respects_limit_and_sort` - DESC order and limit
  - `test_save_score_replace_on_duplicate` - INSERT OR REPLACE behavior
