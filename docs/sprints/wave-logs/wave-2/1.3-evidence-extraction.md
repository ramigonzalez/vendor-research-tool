# Wave Log: Story 1.3 — Structured Evidence Extraction

## Status: Complete

## Summary
Implemented the `extract_evidence` LangGraph node that takes raw search results from Story 1.2 and uses an LLM to extract structured `Evidence` objects for each vendor-requirement pair.

## Files Created
- `app/prompts/extraction.py` — System and user prompt templates for evidence extraction

## Files Modified
- `app/graph/nodes.py` — Appended `_extract_evidence_for_pair()` helper and `extract_evidence()` node
- `tests/test_nodes.py` — Appended `TestExtractEvidence` class with 6 tests

## Key Decisions
- Content from search results is truncated to 2000 characters to stay within LLM context limits
- Maximum 10 results per vendor-requirement pair are sent to the LLM
- Invalid `source_type` values from LLM fall back to `community`
- Empty raw results skip LLM calls entirely (no wasted API calls)
- Extraction failures are logged as warnings and return empty evidence lists (graceful degradation)
